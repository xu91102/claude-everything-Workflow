# 测试与验证

## TDD 工作流

```
1. 理解需求 -> 编写测试
2. 运行测试 -> 确认失败
3. 实现功能 -> 最小化实现
4. 运行测试 -> 确认通过
5. 重构代码 -> 保持测试通过
```

## 测试要求

| 要求 | 标准 |
| --- | --- |
| 覆盖率 | >= 80% |
| 测试先行 | 新功能/bug 修复优先补测试 |
| 关键路径 | 合并前覆盖或明确缺口 |
| 提交前 | 运行与改动范围匹配的验证 |

## 端到端测试

- 技能：`skills/e2e-testing/SKILL.md`，包含 Playwright 目录结构、POM、配置、CI、制品、flake 处理。
- 代理：`e2e-runner`，默认 CLI，MCP 仅用于探索和调试。
- 入口：`/e2e`，复杂或从零搭建时可委派 `e2e-runner`。
