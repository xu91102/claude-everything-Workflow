# 代码规模约束 (CRITICAL)

## 硬性限制

| 约束项 | 限制 | 违反后果 |
|--------|------|----------|
| 单文件行数 | **≤ 600 行** | 必须拆分文件 |
| 单函数/方法 | **≤ 80 行** | 必须提取子函数 |
| 函数参数 | **≤ 5 个** | 必须封装为对象 |
| 嵌套层级 | **≤ 4 层** | 必须提前返回或提取函数 |
| 单行长度 | **≤ 120 字符** | 必须换行 |

## 检查清单

在完成代码修改前，ALWAYS 检查：

- [ ] 文件行数是否超过 600 行
- [ ] 函数行数是否超过 80 行
- [ ] 嵌套是否超过 4 层
- [ ] 参数是否超过 5 个

## 超限处理

当代码超过限制时：

```
文件过长 → 按职责拆分为多个文件
函数过长 → 提取私有函数
参数过多 → 封装为配置对象/DTO
嵌套过深 → 使用早返回模式
```
