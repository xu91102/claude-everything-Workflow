# 禁止事项 (CRITICAL)

## 代码禁止

NEVER 出现以下情况：

| 禁止 | 原因 |
|------|------|
| `any` / `dynamic` 类型 | 破坏类型安全 |
| `@ts-ignore` / `// @ts-nocheck` | 隐藏潜在问题 |
| 函数超过 80 行 | 难以维护和测试 |
| 文件超过 600 行 | 职责不清晰 |
| 嵌套超过 4 层 | 降低可读性 |
| 循环依赖 | 导致耦合 |
| 使用 emoji | 编码问题 |
| `console.log` 调试代码 | 影响性能 |

## 安全禁止

NEVER 出现以下情况：

| 禁止 | 原因 |
|------|------|
| 硬编码密钥/密码 | 安全风险，使用环境变量 |
| SQL 字符串拼接 | SQL 注入风险，使用参数化 |
| 不校验用户输入 | 安全漏洞 |
| 明文存储密码 | 安全风险，使用 hash |
| 提交调试代码 | 影响性能和安全 |

## 提交禁止

NEVER 提交以下内容：

| 禁止 | 原因 |
|------|------|
| `node_modules` 等依赖目录 | 仓库臃肿 |
| `.env` 等敏感配置 | 安全风险 |
| 编译产物 (dist/build) | 应由 CI 生成 |
| 无提交信息的提交 | 难以追溯 |

## 违规检查

发现违规时 MUST：
1. 立即指出问题
2. 提供修复方案
3. 解释为何违规
