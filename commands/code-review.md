# /code-review - 代码审查

对指定代码进行全面审查。

## 使用方式

```
/code-review              # 审查最近修改
/code-review file.ts      # 审查指定文件
/code-review --staged     # 审查暂存区
```

## 审查维度

### 1. 代码质量
- 命名清晰度
- 函数单一职责
- 代码重复度
- 可读性

### 2. 安全性
- 输入验证
- 错误处理
- 敏感信息暴露
- 注入风险

### 3. 性能
- 不必要的计算
- 内存泄漏风险
- 数据库查询效率
- 循环优化

### 4. 最佳实践
- 类型安全
- 错误处理
- 边界条件
- 测试覆盖

## 输出格式

```markdown
# 代码审查报告

## 严重问题 (必须修复)
- [问题描述及位置]

## 建议改进 (推荐)
- [优化建议]

## 优点
- [做得好的地方]

## 总结
[总体评价]

审查结果: [通过/需修改/拒绝]
```

## 严重级别

| 级别 | 说明 | 行为 |
|------|------|------|
| 严重 | 必须修复 | 阻止合并 |
| 警告 | 建议修复 | 可以合并 |
| 建议 | 可选优化 | 自行决定 |
