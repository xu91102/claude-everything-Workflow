# /plan - 实施计划

在编写代码前创建全面的实施计划。

## 使用场景

- 开始新功能
- 重大架构变更
- 复杂重构
- 多文件/组件变更
- 需求不明确

## 流程

1. **重述需求** - 明确要构建的内容
2. **识别风险** - 发现潜在问题和阻碍
3. **分阶段计划** - 将实现分解为步骤
4. **等待确认** - 必须获得用户批准

## 输出格式

```markdown
# 实施计划: [功能名称]

## 需求重述
- [需求点1]
- [需求点2]

## 实现阶段

### 阶段 1: [名称]
- 步骤 1.1
- 步骤 1.2

### 阶段 2: [名称]
- 步骤 2.1
- 步骤 2.2

## 依赖项
- [依赖1]
- [依赖2]

## 风险评估
- 高: [风险描述]
- 中: [风险描述]
- 低: [风险描述]

## 复杂度估算
- 后端: X 小时
- 前端: X 小时
- 测试: X 小时

**等待确认**: 继续执行此计划? (yes/no/modify)
```

## 重要提示

**关键**: 在用户明确确认前，不会编写任何代码。

如需修改，回复：
- "modify: [修改内容]"
- "跳过阶段2，先做阶段3"
