# /learn - 提取可复用模式

在会话中解决了非平凡问题后，运行此命令提取模式。

## 使用方式

```
/learn              # 分析当前会话，提取模式
/learn --preview    # 预览但不保存
```

## 检测的模式类型

1. **错误解决模式** - 如何修复特定错误
2. **调试技巧** - 有效的调试方法
3. **变通方案** - 库/API 的变通方法
4. **项目特定模式** - 项目约定和规范

## 输出格式

保存到 `skills/learned/[pattern-name].md`：

```markdown
---
name: pattern-name
extracted: YYYY-MM-DD
confidence: 0.7
domain: [领域]
---

# [描述性名称]

## 问题
[此模式解决什么问题]

## 解决方案
[模式/技巧/方案]

## 示例
[代码示例]

## 触发条件
[何时激活]
```

## 流程

1. 分析会话中可提取的模式
2. 识别最有价值的洞察
3. 生成技能文件草稿
4. 确认后保存
